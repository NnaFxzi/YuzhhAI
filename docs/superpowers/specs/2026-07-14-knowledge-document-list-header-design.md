# 资料文档列表栏设计

## 目标

将“当前文档 · 数量 + 搜索文档”这一行改成清晰的文件列表栏：顶部保留上传文件，列表栏展示“文件列表”和数量，搜索在文档较多时直接显示、文档较少时通过搜索图标按需展开，回收站移到面板左下角。

## 设计

```text
文件列表  3                                  [⌕] [上传文件]
```

- 活动文档视图的标题为“文件列表”，回收站视图的标题为“回收站”。
- 数量使用紧凑徽标显示，不再使用“当前文档 · 3 个”的长文本。
- 少于 8 个文档时只显示搜索图标；点击后展开名称搜索输入框。
- 8 个或以上文档，或已有查询词时，直接显示搜索输入框。
- 回收站作为底部低频入口固定在面板左下角，不再出现在顶部切换控件中。
- 上传文件继续作为顶部主要按钮，沿用现有上传回调和禁用状态。
- 文件行、状态图标弹窗、AI 知识提取按钮、详情面板和删除/恢复逻辑不变。

## 范围

### 包含

- 修改资料文档面板的列表标题栏布局。
- 增加搜索图标展开/关闭状态。
- 将回收站切换入口移动到面板底部。
- 增加中文和英文的列表标题、数量文案。
- 增加静态渲染测试，覆盖列表标题、数量、搜索按需显示和回收站底部入口。

### 不包含

- 不修改文档导入、解析、本地索引、AI 提取或 IPC 逻辑。
- 不修改外层知识库页面和 AI 知识视图。
- 不增加状态筛选、排序或批量操作。

## 验收标准

1. 顶部仍然可以看到上传文件按钮。
2. 顶部不再出现“当前文档 / 回收站”分段切换。
3. 文件列表栏显示标题和数量徽标。
4. 少量文档时显示搜索图标，点击后可以展开搜索框。
5. 回收站入口固定在资料文档面板左下角，并能正常切换视图。
6. 现有文件行状态图标和 AI 提取操作保持可用。
