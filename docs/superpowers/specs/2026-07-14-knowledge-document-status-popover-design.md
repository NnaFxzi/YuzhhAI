# 文档状态面板设计

## 目标

在紧凑的资料文档列表中，让用户可以可靠地点击解析、本地索引和 AI 提取图标，查看完整状态信息，并避免状态面板被列表滚动容器裁剪。

## 方案

每个文档行保留三个状态图标作为独立按钮。点击后，将状态面板通过 React portal 渲染到 `document.body`，使用触发按钮的 `getBoundingClientRect()` 计算位置：默认显示在按钮下方；当下方空间不足时改为显示在按钮上方，并限制在视口范围内。面板包含状态名称、当前状态、进度/更新时间以及已有的重试、取消操作。

面板支持点击外部关闭、Escape 关闭和触发按钮的键盘焦点回收。触发按钮通过 `aria-expanded`、`aria-controls` 和状态说明提供可访问反馈；打开状态使用主题色边框与浅色背景突出显示。文档列表仍保持紧凑，不因状态详情展开而改变行高。

## 不在范围内

- 不修改知识库状态数据、轮询或 IPC 协议。
- 不改变文档详情抽屉和 AI 提取授权流程。
- 不把状态详情常驻到每一行，避免降低多文档浏览密度。

## 验收标准

- 点击任一状态图标后，完整面板可见，不被列表、滚动区域或页面边界裁剪。
- 面板在列表靠近底部时自动向上定位。
- 点击外部或按 Escape 后面板关闭，触发按钮恢复焦点。
- 解析失败、本地索引失败、AI 提取进行中等现有状态内容仍然正确显示。
- 相关 renderer 测试、定向 lint 和 production build 通过。
