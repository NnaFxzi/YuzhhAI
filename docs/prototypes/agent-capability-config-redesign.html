编辑工作区 Agent

这里的修改只保存到当前工作空间，并用于新的对话和下一次任务执行。

基本信息

编辑这个 Agent 在当前工作空间里的名称、头像、职责说明和执行规范。

执行规范
把稳定 Agent 需要遵守的做事方式写清楚，减少每次对话里的重复解释。
定义判断顺序，避免跳步和直接下结论。
说明哪些资料可作为判断依据。
固定结构，方便阅读和后续 Agent 接力。
限制不能猜、不能承诺、需要人工确认的情况。

示例与校验

用典型任务检查这个 Agent 是否按固定结构回答，并遵守缺失信息和人工确认边界。

1

高意向询盘

信息较完整，Agent 应该给出明确优先级和下一步动作。

示例输入

用户会这样提问
保留真实业务表达，不需要写得像提示词。

期望输出

Agent 应该稳定交付
稳定性检查
1 输出必须包含客户优先级、判断依据、缺失信息和下一步动作。
2 目标价格缺失时只能标记待补充，不能自行推测。
3 涉及报价、交期或承诺时提示人工确认。
2

信息不足

资料不够时，Agent 应该先列待补充信息，而不是直接判断。

示例输入

用户会这样提问
这个样例用来训练 Agent 不要在信息不足时猜测价格或优先级。

期望输出

Agent 应该稳定交付
稳定性检查
1 必须明确标记信息不足或待判断。
2 不能给出价格、交期或确定优先级。
3 必须给出下一步需要客户补充的资料清单。
3

需要人工确认

涉及报价、交期、外发承诺时，Agent 应该主动标记人工确认。

示例输入

用户会这样提问
这个样例用来训练 Agent 主动识别外发风险和人工审批边界。

期望输出

Agent 应该稳定交付
稳定性检查
1 必须标记人工确认，不能直接生成正式承诺。
2 不能承诺价格、交期、产能或合作结果。
3 必须说明需要技术和销售确认后再外发。
